## How Email Verification Works

### Registration Flow
1. User registers → User created in DB with `verified: false`
2. Verification token generated (JWT with user email)
3. Verification email sent (with link to click)
4. User clicks link → Email verified and set to `true`
5. User can now login

### What Changed
```javascript
// BEFORE: User created without explicit verified field
const user = await userModel.create({ username, email, password })

// AFTER: User created with verified: false explicitly
const user = await userModel.create({ 
    username, 
    email, 
    password,
    verified: false
})
```

## Gmail Configuration (Step-by-Step)

### Step 1: Enable 2-Factor Authentication
1. Go to https://myaccount.google.com/security
2. Find "2-Step Verification" and enable it
3. Follow the prompts

### Step 2: Generate App Password
1. Go to https://myaccount.google.com/apppasswords
2. Select "Mail" and "Windows Computer" (or your device)
3. Google will generate a 16-character password
4. Copy this password (without spaces)

### Step 3: Configure .env File
Create or update `Backend/.env`:

```env
# Gmail Configuration
GOOGLE_USER=your-email@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
```

**Important**: Copy the exact password from Google (it includes spaces), and make sure:
- `GOOGLE_USER` matches your Gmail address exactly
- `GMAIL_APP_PASSWORD` is the 16-char password generated by Google
- Both variables are on separate lines in .env

### Step 4: Verify Configuration
When you start the backend, you should see:
```
✅ Using basic auth for Gmail (App Password)
✅ Email transporter is ready to send emails
```

If you see errors, check:
- Is `GOOGLE_USER` set in .env?
- Is `GMAIL_APP_PASSWORD` (or OAuth2 credentials) configured?
- Did you generate the App Password correctly?
- Is 2-Factor Authentication enabled on Gmail?

## Troubleshooting

### Issue: "Email not sent" with no logs
**Solution**: Check that environment variables are set:
```bash
# In your Backend/.env file, verify these exist:
GOOGLE_USER=your-email@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx
```

### Issue: "Account security issue" error
**Solution**: It means the app password is invalid or Gmail didn't generate it correctly:
1. Delete the app password from https://myaccount.google.com/apppasswords
2. Generate a new one
3. Update .env with the new password

### Issue: `verified` field is null or missing
**Solution**: The field should always be initialized to `false`. If missing:
1. Delete existing test users from MongoDB
2. Register new users (they will have `verified: false`)
3. Or manually update in MongoDB: `db.users.updateMany({}, { $set: { verified: false } })`

### Issue: User can login without email verification
**Solution**: Check auth.controller.js login() function. It should have:
```javascript
if (!user.verified) {
    return res.status(400).json({
        message: "Please verify your email before logging in",
        success: false,
        err: "Email not verified"
    })
}
```

### Issue: Backend logs show email verification URL issue
**Solution**: Make sure backend URL is correct in .env:
```env
BACKEND_URL=http://localhost:5000  # for development
BACKEND_URL=https://api.example.com  # for production
```

## OAuth2 Configuration (Advanced - Optional)

If you want to use OAuth2 instead of App Password:

```env
GOOGLE_CLIENT_ID=your_client_id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your_client_secret
GOOGLE_REFRESH_TOKEN=your_refresh_token
GOOGLE_ACCESS_TOKEN=your_access_token
```

The system will automatically use OAuth2 if these are configured, otherwise it falls back to App Password.
